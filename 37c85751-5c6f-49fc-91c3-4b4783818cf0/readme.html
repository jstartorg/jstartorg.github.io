Overview

  • Jumpstart's information infrastructure is a hybrid of On-Premise, IaaS, and SaaS resources hosting a variety of User Services to support Jumpstart's mission and vision. It is primarily developed, supported, and maintained by our Technology team and an external MSP as backup.
  • The source of truth for most of our data is Salesforce, the only exception being financial data that is stored in Paycom and Intacct.
  • Authentication and Authorization are handled by Active Directory throughout the vast majority of systems, either natively, through an Azure connection, or bespoke through Power Automate flows. There are very systems that have not been accounted for this way, the chief being the PER System, a sunsetting custom application.
  • Hardware provided to staff are standardized to lightweight mid-range Lenovo ThinkPad (eg T490, T14) models, with older models having at least 8Gb of memory and all new units having 16Gb, with Windows 10 installed and necessary software and policies deployed through Group Policy.
  • Each office is integrated into the overall network through a Meraki MX64 firewall's AutoVPN feature, except for our AWS environment which uses a Meraki-vMX-S EC2; each firewall is also a VPN endpoint for staff to connect to when working remotely.
  • Communication is handled through Office 365's services including Exchange for email and Teams for remote meetings. We provide one Zoom account to each department for the cases when external partners are unable to use Teams or a particular feature of Zoom is required. The Zoom accounts are NOT to be used to meet with children or families as we cannot implement the same level of policy controls that Teams provides at our license level.
  • File storage and sharing can be done through a traditional file share called the J Drive, divided by department or workgroups - but this is being encouraged only for archiving or historical reference; through OneDrive/SharePoint as integrated in Windows and the Office 365 environment - this is the preferred way for most work; and finally staff can technically use Google Drive which was implemented purely because so many staff already were using personal accounts, but it is strongly discouraged as nonstandard and unintegrated with the rest of the Office 365 tools.
  • Work management for the Technology team is done through Salesforce Cases for requests and LanSweeper for hardware resource tracking.
  • Backups for data across systems takes advantage of several different tools, most importantly OwnBackup for Salesforce, Veeam for on-premise servers, Lifecycle snapshot handling for AWS EC2s, Active Backup for 365 on JS-BOS-NAS for OneDrive and C-level email accounts, and CloudBerry Backup to both on-premise NAS and AWS Glacier for file-level backups.
  • Domain name management is handled internally by Windows domain controllers and externally through DNSMadeEasy.
  • Reporting across systems is done with Power BI using the desktop software as well as dataflows drawn from Salesforce, the Areas of Concern/Data warehouse, and Parquet/CSV files on the J Drive.
  • Security certificates for website SSL and server connections are provided by LetsEncrypt through CertifyTheWeb or Acme (certbot) scripts on the server.
  • Monitoring is handled by both internal and external tools - internally by UptimeKuma on DockerHost1 and externally by Uptime Robot; these focus on website and server availability, that compliments the built-in self-monitoring of platforms such as User Services/meraki and Office 365
  • Remote access is supported by multiple Meraki ClientVPN endpoints as well as a SoftEther connection for failover/SSTP compatibility; we also have two remote desktop systems, one for general use at access host on AWS and a workstation for the Technology team at JS-AZ-Apps host on Azure.

Structure of Documentation

User Services - all the websites where end-users would go
System Services - large scope of work sysytems like Salesforce and Active Directory

JSI Model

Jumpstart Systems Interconnection

Layer Name Description
7 🕹 Service Describes the broad what and why, as well as any user-facing use/access notes
6 📽 Software The software, with details about licensing, hw requirements, install notes
5 🔗 Auth The way users are managed, what kind of groups are there, permissions stuff
4 🚚 ISP Account numbers, site to use to check for outages, vendor connection details
3 ⚡ Network Security/firewall/routing rules or system, especially with regard to connecting multiple offices or data centers
2 🛠 Framework The layer that things actually run on, like virtual machine software, hosting platform
1 💻 Server Physical or hosted hardware ie Hyper-V host or AWS instance

User Types

Name Description
Shepard 🐑 Person/Role resposible for overseeing particular layers/services. ex: Data specialist is resposible for the Data Warehouse
End User 😀 Various users, Staff/Corps/External, who interact with our Services and Software layers
