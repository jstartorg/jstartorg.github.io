Salesforce Object Query Language to search your organization’s Salesforce data for specific information. SOQL is similar to the SELECT statement in the widely used Structured Query Language (SQL) but is designed specifically for Salesforce data

Why

  • Provides access to more data than that visible in the UI
  • Allows for more movement up and down relationships
  • More robust and targeted extracts

Why Learn SOQL

  • No need for custom report types
  • More custom criteria/organization for extracts than Reports
  • No need for separate applications to create extracts
  • Tools like Salesforce Inspector (free chrome extension) or the Development Console give you access to the whole databse in a couple of clicks
    • data loader (sf)
    • workbench → his preference/most user friendly when learning
  • Keystrokes are always faster than clicks. SOQL gives you the ability to scan using the keyboard....

Where SOQL can be used

  • Quickly comparing records/reviewing specific data points
  • Create a quick extract to ping a specified group of records
  • Review specific group of parameters and outcomes across extensive dataset
  • Conga Solutions
  • complex data loader extracts
  • Communicating with developers
  • Understanding code, flows, processes and other SFDC functionality (can become more fluent in other areas)

Basis of SOQL

  • Select function
    • Signifies a list of outputs (fields or data points) that the query should return
    • Data points can be found through relationships (’_r’ instead of ‘_c’)
    • Example → SELECT Email, SELECT id, SELECT Account. MailingStreet, SELECT count()
  • From function
    • Choose an object to pull data from
    • More objects than just those featured in the UI are available
    • Groups (Queues, Public Groups, etc.)
    • Feed Tables
    • Examples: FROM Account, FROM Contact, FROM AccountShare
  • Where function
    • Used to filter a table using another table/list (limit 2)
    • Cannot exceed 4000 characters
    • Can Use AND, OR and () to provide advanced logic
    • IN/NOT IN
  • Order By function
    • Ascending/Descending

Child to Parent

  • 5 Levels
  • 36 Relationships

Parent to Child

  • 1 Level
  • 20 Relationships

Limitations

  • SOQL Limits on Objects → a list of limitations
  • Salesforce Object Query Language (SOQL) document → has all functions

r = relationship to field (rather than field itself)
