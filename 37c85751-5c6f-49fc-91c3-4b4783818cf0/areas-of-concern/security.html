General Principals

  • All Tech staff should use a normal user account for all daily actions that has no special permissions
  • Any permissions they need for their role should be housed on a "name"admin account they can use as needed
  • The less permissions for the least amount of time is the ideal way access should be granted
  • deletion of data should be strictly limited and audit trails should be activated in any system that supports them
  • permission to create shared folders, groups, or any other resource that is meant for multiple users should be very limited as well
    • partly this help prevent accidental overreach, like the site manager who created a Team called "Corps Members" just for her site before we locked down Team creation

Staff Security Levels

Data Reporting Specialist

Security level

  • large scope of relevance defined by particular projects, but should most often be time bound and the long term permissions should be assigned to the report.admin user once report development is done

Directory of Technology

Security Level

Security Level

  • all data in all systems - ONLY through their admin login, to be clear
  • should have all accounts protected by MFA

Senior Salesforce Administrator

Security Level

Security Level

  • permissions to
    • access everything, including PII, in Salesforce and supporting systems
    • edit records and automations within Salesforce and in many supporting systems (ie Power Automate)

Senior Support Manager

Security Level

Security Level

  • permission to
    • reset passwords
    • add self to resources on a temporary basis to help with tickets
      • Teams
      • department shares
      • email inboxes
    • add and remove licenses from users but should not handle purchasing in most cases

Support Specialist

Security level

Security level

  • relatively low permissions
  • can reset AD passwords
  • cannot give self access to inboxes/folders/etc
  • can know the local PC admin login information for helping with software installs

System Administrator

Security Level

Security Level

  • nearly everything outside of Salesforce
  • potential PII access through forms submissions
  • assist the director with purchasing and distributing software licenses
  • can create Group Policy that affect all PCs and users in domain
  • can create Teams/AD Groups/SharePoint sites
