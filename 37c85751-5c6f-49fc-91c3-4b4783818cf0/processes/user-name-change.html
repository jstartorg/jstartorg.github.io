Description

The first and most important changes happen in Active Directory, and for the most part propagate out as systems synchronize.

  • Active Directory is the Source of Truth for user accounts
  • Office 365/Azure Active Directory identities gets synced through Microsoft's own software approximately every 20 minutes
    • AD Group memberships get synced at the same time, but a different process runs every two hours to sync AD Group memberships to Office 365 Groups memberships
    • Similarly another process updates Azure App Registrations to give login/use permission for various SSOed services
  • Google Suite will pick up the new username automatically

Manual updates

  • Salesforce may need the Federation ID field on the User record #todo need to verify
  • MyJStart needs the Username field updated which must be done through the Admin Dashboard
  • LastPass blocks changes to email addresses so they must be added to the allow policy
  • Areas of Concern/Data warehouse entries for course progress and surveys must be updated

Updates not needed

  • Easygenerator does not directly manage users, instead relying on the Azure SSO connection, and its output is saved to the warehouse
  • Cognito Forms does not need any updates, but any un-submitted data (member forms, surveys, etc) will need to be restarted to make sure when they do submit to the warehouse or Salesforce they are correctly associated (already submitted data will be updated by the rest of the procedure)

LastPass

Email address changes are blocked by policy (so people can't change them to a non-Jumpstart address and prevent access later/make things confusing). In order to change their login/email they must be added to the Prohibit account email change Allow List, then they can confirm the change. After that they should be taken back out of the allow list.

Data Warehouse

Updating the warehouse requires running UPDATE queries directly, so I am working on a Windmill flow to handle it, but in essence you can cover it with three queries:

UPDATE warehouse SET email_id = 'NEW_USERNAME' WHERE email_id = 'OLD_USERNAME';
UPDATE warehouse SET data = jsonb_set(data, '{actor,mbox}', '"mailto:NEW_USERNAME"', false) WHERE email_id = 'NEW_USERNAME' AND origin = 'course';
UPDATE warehouse SET data = jsonb_set(data, '{Email}', '"NEW_USERNAME"', false) WHERE email_id = 'NEW_USERNAME' AND origin LIKE '%survey%';
