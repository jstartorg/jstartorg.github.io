Content

Still managed by MarCom, but structured more dynamically with the idea being any piece of information should exist in just one record/post/page/POD (see below) and be re-used elsewhere. So far this means:

  • posts = news/updates
  • sites = Higher Ed partners
  • states = states
  • people = leaders/staff/featured external figures
  • supporters = donors or programming partners
  • content = published materials like activity guides or community stories

Media is also managed differently, with more consistent structure, required alt-text descriptions, categories, and naming conventions. PDFs are managed as media, with generated thumbnails rather than manual screenshotting. Publishing all content is driven by dynamic queries on categories or requesting page, very little should ever be directly embedded and lists should never be manually built but rather query driven.

SEO is handled by the AIOSEO plugin which will make clear what should be added to pages to help with search engine presence.

Plugins/Integrations

  • WP Code Lite - supporting functions and settings
  • Otter Blocks - Gutenberg blocks (used lightly)
  • Theme Switcha - to preview themes as an admin
  • Media Assistant Lite - much better media management, media category use
