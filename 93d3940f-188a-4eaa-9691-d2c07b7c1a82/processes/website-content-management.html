Description

#marcom #website

The main website is used for marketing and communications purposes, supporting four primary goals - engaging donors, recruiting Corps Members, reconnecting with alumni, and platforming our advocacy efforts. Starting from a fresh build of WordPress, the site has been customized and expanded to support greater automation and management across those different goals. The most crucial drivers of this are two-fold - relying heavily on Categories shared across posts, pages, media, and custom content to relate and display relevant information; and the use of custom content types to standardize data entry and management.

Categories

The same list of Categories is shared between all units of content. Some of the categories are just for simple organization or to make sure things like search engines or related content suggestions have relationships to use, but some Categories are essential to the functioning of the site. In general try to use a decent number of categories on every piece of content in the site as it will help you identify, reuse, and relate things going forward.

Newsflash

The Category with the broadest possible reach, showing up in all listings of Posts across audiences.

All Sites and All States

In keeping with greater automation, each Site and State gets its own page, where most of the content is just pulled from relevant pools rather than requiring a lot of hands on updates. So for instance, if Jumpstart is launching a new brand, by just selecting the "All States" and "All Sites" categories then all of the Site and State Pages will automatically pull that announcement in as if it were directly related to that audience.

Media

  1. First check if that media is already present – eg Nigel Barker’s profile picture has already been uploaded and is in the library
  2. Upload just one, high-quality copy of a piece – most uploads should be around 1000 pixels wide at most if available, not huge but not the more medium size 600-800 pixel range. Wordpress will generate smaller views of large pictures, but if you ever want to feature some image in a big way it can’t make a small photo bigger
    1. if the media is a logo or some sort of design element, the best format to use is SVG, which will scale and resize cleanly
    2. for images used as banners or other page-spanning layout roles a larger image may be appropriate - the current standard screen width for desktops and laptops is 1920 pixels, though many laptops do still operate at 768 or 1400 pixels wide. Hi-res monitors commonly support up to 2560 pixels, but 1920 is close enough that the builtin stretching that browsers do should not make a huge difference
  3. Add alt text – this is a requirement, it is needed for both SEO and accessibility purposes. It doesn’t need to be complicated, like with the Nigel Barker example I literally just made it “Profile picture of Nigel Barker”, but that is enough to give people an idea of what is going on
  4. Add a clear title – the title field will help you/others find the photo later, and helps keeps things more readable when viewing the media library. That again can be really simple, I named the Barker profile just “Nigel-Barker”
  5. Select meaningful categories – this is huge, having the right categories will not only help with management of pictures but also when different galleries or listings are made they are most often driven by categories; they also make step #1 much easier as well

    Include-Page Shortcode

    A simple shortcode to allow the transclusion of pages. Especially useful for yearly pages like the Annual Report, where a new page is made yearly but it is ideal to keep the previous years. The use of it is very simple, just create a paragraph block with the text

    [include-page pageid="#####"]

    where the #### is replaced with the page ID of whatever content you want to show. To get that ID you can go to the page you'd like to include and click Edit Page, then you can pull the ID from that URL

    https://next.jstart.org/wp-admin/post.php?post=######&action=edit

    This does not need to be the total content of a page - you can use the same chunk of content from the source page and add on further content as you'd like.

DEPRECATED: Site Sections

The "Site Section: ABCD" categories are the most critical as they drive not only relationships but also display styles on page. Each Site Section has an associated header and sidebar color scheme in the stylesheet. The Site Sections are meant to map pretty closely to the primary goals, with a slight twist:

  • Site Section: Advocacy - PGR focused content, with some shades to alumni
  • Site Section: Organization - main content, like org leadership, mission, and even the donate page, just to keep the brand 100% clear
  • Site Section: Program - used for program related things like programming descriptions, things having to do with recruitment, content for HEPs
  • Site Section: Read for the Record - pretty straightforward, but generally can be looked at as the distinctly development side of things
    Not only will that drive visual styles, but elements on the page (Lazy Blocks like Post List from Page Categories, Content List by Page Category, Job List by Page Category) will use the Categories to show related info, and the more consistent the Site Sections are used the better those blocks will work.
