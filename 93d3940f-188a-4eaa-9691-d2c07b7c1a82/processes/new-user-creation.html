Description

New user creation is driven from Salesforce from either Contact or Person (Lead) records. Corps Members recorded as Leads and are the bulk of new accounts, and those are driven by an automated process that is triggered by the Criminal History Check Process reaching a certain point.

All other accounts are generated from a manual approval in Salesforce on the Contact records using the Active Directory field set:

  1. Active Directory Username will be blank until the user is created and the username is synced back into Salesforce
  2. Active Directory Update Required is updated by automatic processes to try to help indicate when a change has occurred that may require an update - like a new record being added, a change of department, a change of site, etc. Is cleared automatically when a new account is created, otherwise can be cleared manually if other updates have been made (or none were needed)
  3. Active Directory Account Type specifies the set of groups the new user will have, as well as the OU they will be stored in. The options are "Employee (Non-Site)", "Higher Education Partner Staff", "Site Manager", "Program Assistant" and is required before a new account can be created
  4. Active Directory MyJStart ID is a site-based field and will only appear if the Primary Affiliation of a Contact is a Jumpstart site Account, and is automatically calculated but shown for your information
  5. Active Directory Profile is a link to a LanSweeper search for the person's last name (the only way to specify a person currently). If they have an AD account this field will populate, allowing direct viewing and manipulation of the account with LanSweeper's scripts and tools
  6. Active Directory Push Sync is the key field when creating new accounts - checking this box (if all other necessary data has been filled in) will trigger a Power Automate/Windmill flow that creates the new user account and emails the personal account of the person with a welcome email
  7. Active Directory Department is for when the individual is a staff member, whether non-site or site-based, and makes sure they are put in the correct AD groups
  8. Active Directory Site Code is a site-based field and will only appear if the Primary Affiliation of a Contact is a Jumpstart site Account, and is automatically calculated but shown for your information

Intake

The data in Salesforce comes from one of two places - either from the MyJStart contact manager page for Higher Education Partner Staff, or a completed Personnel Form for everyone else. Both tools find or create a Contact record and populate it with initial information about what the account will need.

Account Types

Employee (Non-Site)

These users are the most straightforward - all that needs to be selected is their Department.

Consultants

Consultants are a special case, with their own process elements, but the fundamentals of their account creation is the same as non-site employees. A form should be submitted for them, detailing what department they will be working with, and then their account can be created as normal.

Site-Based

Site-based users need to have their Primary Affiliation set to the Account of the site they will be working with. This ensures that they will end up with the correct Access-Role- group and any other specifics associated with that site.

Site Manager

Site managers will end up with two Affiliations at least, one for their site and one for Jumpstart itself, and both should have the Status "Current".

Higher Education Partner Staff

They do not need any special settings beyond their Primary Affiliation. Some may already have Contacts and Households if they have ever donated to Jumpstart before, for instance.

Program Assistants

Also only need the Primary Affiliation. A Program Assistant is broadly any role taken on by a student that isn't working as a Corps Member.
