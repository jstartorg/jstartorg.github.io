Teams

Description

The Teams phone system is a full replacement for a traditional VOIP system/physical phone. It maps a real phone number to an Office 365 user or resource account (think shared inbox for phones). The resource accounts can then either map to call queues or auto-attendants. NOTE: SMS texting is not included in this phone system.

Not all staff get a direct dial phoneline. It is by request/need only, as there is a cost for each line. Generally people who have consistent need to interact by phone directly with external parties are given a number - many (but not all) Development staff, some Talent Management, and Policy and Government Relations. All departments have a shared line (technically a Call Queue, explained below), that can be used by any department member to make phone calls out. It also receives phone calls and can be configured for round-robin, simultaneous, and other call allocation modes.

Numbers

Phone numbers can either be ported in to Teams or requested from Microsoft. We already have a block of numbers for the old phone system, and largely they are just moved around as staff come and go. The geographic prefix is not actually relevant to the function of the line, but many times we make the effort to get staff a number relevant to the office they are closest to. It is also good practice to keep phone numbers in a given department, so that if someone calls an old phone number they have they might get someone still relevant.

Line Types

User

This is the most traditional phone system setup. One person gets one phone number, and from their Teams client (both on the laptop and on mobile devices) they can make and receive calls. In order for a user to get a phone number though they must have two different licenses - a Phone System license (provided to all employees through the E5 license, only purchased separately for affiliate staff if there is some clear reason, so far there has not been), as well as a Dialing Plan. They must have the former to use Teams phones at all, and the later is required to have a personal line.

Another requirement is that their Azure user account must have Enterprise Voice enabled - but there is no way to enable that through the Admin user interface. Instead you have to use the PowerShell commandlets for Teams to up the user to set EnterpriseVoiceEnabled to $true.

Only after all that is set up can you select a number for a user in the Teams Admin portal. You can also set up details like call forwarding, voicemail handling, and other options.

Resource Account Lines

Lines meant to be used by multiple people still have to be associated to a single user-like object called a Resource Account. This is similar to a shared inbox in Exchange. When you buy licenses for actual users, Microsoft adds a certain number of available resouce account licenses to the organization, and we have yet to run out of those. Once a number is associated with a resource account, the account itself must be associated with one of two "applications".

Call Queue

A Call Queue is meant for distributing calls among the staff in a group (like a department, but doesn't have to be). It can be set up using a true AD group or by manually selecting user by user. Each of the users in a call queue must have a Phone System license in order to participate - at Jumpstart that is provided by the E5 license purchased for all employees. Distributing incoming calls can be done a few different ways, such as round robin, last active first, all at once, and maybe a few more. Generally whichever group will be affected get to choose, and if it is round robin they can define what the order of ringing is.

Auto-Attendant

This is a traditional phone menu system. It is associated with a resource account, but does not need to point to any individual user unless desired. It is also possible to let it do voice or type searching of a directory on top of the normal multi-level number defined menu setup. There is an auto-attendant for several offices and at one point in time for Directors of Programming, though that is likely not in use at this point.
