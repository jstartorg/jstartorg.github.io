When someone asks you to create a Report, ask

  • Intended purpose? Why?
  • Who will use? How often?
  • Do you have a mockup?
  • What information (Objects)?
  • What fields?
  • What timeframe?
  • Would a chart or conditional highlighting be helpful?
  • What is your confidence in the underlying data? Issues?

Reports Tab & Folder

  • Categories
  • Search
  • Favorites
    • Can favorite reports, dashboards, folders, list views (embedded in favorites tab)
    • Each user can have up to 200 favorited items
  • Fields Displayed
    • Can add column(s) to display
    • Gear top right → Select fields to display
    • Can sort list view by columns/fields displayed
  • Actions
    • Dropdown next to report name in folder (run, edit, add to dashboard, favorite, move, etc)

Sample reports in Salesforce Classic → can run & save and then switch back to Lightening

Each user can subscribe to 5 reports
2 subscription types in Classic → can subscribe & schedule runs, not limited to user (and then can switch back to Lightening; but the reports the person would get would be Classic)

  • Workaround = if you want 6 reports, put them in 1 dashboard & subscribe to dashboard

Switch to classic if reorganizing a number of reports/folders (can drag and drop)

In Line Editing → Needs to be turned on! (in line edit text, currency, picklist, etc.) - edit directly when editing report

  • This is either turned on for everyone in org or off (not yet a profile permission for just some users)

Group Rows by Field (can be done in main report view, without edit view)

Protect reports → put them into folders that are Read Only

  • Other users can save as to create own copy

Contact or Lead reports can be added to Campaign (dropdown arrow where you go to edit)

Folders → all reports and dashboards should be saved in folders

  • Private Reports (default)
  • Public Reports
  • Nest custom Folders 4 levels deep (Lightening feature!)
    • Permissions are all controlled at the top level
    • Label clearly (ex. Asia Pacific Sales, Asia Pacific Marketing)
  • Share ONLY from Root Level (subfolders inherit)

Folder Sharing

  • Sharing is explicit
  • Share with Users, Public groups, Roles, Roles & Subordinates
    • More dynamic to share with Public groups/Roles
  • View, Edit, Manage
    • Should use View most often! (can see, run, and save copy of report; can’t edit and break it directly)
  • Nested Folders inherit sharing

By default, new Folders are private → need to share Folder(s) with users/groups

  • Need to actually click Share button!

Group Columns (horizontal grouping)

3 of 4 Report Formats

  • Tabular
    • No groups, just rows and columns!
    • Great for exportting a list
    • Feed tables on dashboards
  • Summary
    • Group rows up to 3x
    • Add summary columns & formulas feed any dashboard
    • Feed any Dashboard Component
  • Matrix
    • Groups rows & columns up to 2x each
    • Add Summary Columns & Formulas

Report Filters

  • Advanced Filters (can have up to 20)
  • Relative Date Values (words you can use in report filters/list views like “last week” “this week” “this quarter)
    • Not default date filter that is there (add another date column to filter down below → relative date option here)
  • Filter Logic (ands & ors)
  • Cross Filters

Relative Date Values (look up in Help! very useful for reporting)

  • Filter by Created Date → Use relative date (”this week” will always be this week, whenever report is run)
  • Best to use Equals or Not Equals (ex. equals “last 88 days”)

Filter Logic (ex. (1 AND 2) OR 3)
(by default, multiple filters are and - x and x and x)

Cross Filters

  • Create report on one type of data (ex. Accounts) - can filter on report with filters on another object
  • Need to have a Parent Object that has a Child Object (ex. Expense Reports & Line Items - show me expense reports without line items)
  • Can create up to 3 cross filters per report
  • Ex. Accounts without Cases (report on customers who have submitted a case)

Charts

  • Available w/ at least 1 grouping
  • Add while editing or running Report
  • 8 Types, each with own Properties
  • Stacked → have to have at least 2 groupings(?)
