As admins and implementation consultants, it’s easy to get on a roll and set things up in Lightning either based on the ‘out of the box’ formats or based on what you think looks good. But to really maximize the User Experience, we have got to learn to place ourselves in the seat of

Know your users

  • Figure out your users' personas (roles within the system)
    • Avoid having too many profiles, apps, page layouts
    • Group by key functions
      • Object permissions
      • System permissions
      • Put the critical information they need to know/steps they need to take in the system right in front of them
    • Watch them as they go through processes (sometimes people don’t use it the way you built it)
    • When somebody asks for a new field etc, ask how they’re going to use it, and if they don’t know or say they may need it later, say then the data will be garbage so no. Could also put history tracking and analyze how much it’s filled out. Could also take it off the page and wait until somebody complains about it.

Apps

  • One app per persona (turn off the others unless they’re hybrid users)
  • Configure default tabs that are foundational for that app (and then train users to personalize)
  • Order the tabs in the order of the business process
  • Search returns what's in the tabs first

Lightning pages and page layouts

  • Conditionally display fields and sections
  • Don’t want unnecesary fields to send people down a rabbit hole or confuse people with things they don’t need
  • Different sizes of sections for different profiles
  • Rich text component as guidance when user is within first N days
  • Dynamic actions and forms (will be available in Accounts, Contacts, and Opportunities in Winter 23, later for Leads and Cases)

Reports & Dashboards & List Views

  • Limit view and edit access (they suggest admins only)
  • If a report hasn’t been run in X time, move it to another folder, wait longer, and then delete it if never used.

Home Page

  • Add dashboards

Utility bar

  • Can pop up a screen from any page, doesn’t take up tabs
  • App specific
  • Add easter eggs for fun

Security

  • Muting permission sets (look into this more)
  • Someone noted a bug with permission set groups where if you add a new permission set to the group, the users don’t get the permission, you end up having to add it to the user

Leadership support

  • If leaders don’t care, users don’t care
  • If someone can go 2 months without logging in, that’s an issue with leadership not administration. Because then you can’t trust the data that’s in the system, and leadership is either not looking at or doesn’t care about/understand the data
  • After a certain amount of time not logging in, freeze user

Automations

  • Screen flows
  • Next best actions
  • Notifications (emails, scheduled reports, chatter notifications)

Communication of changes

  • Use multiple ways bc people consume info in different ways
  • Can use in-app guidance and lightning components for small changes
