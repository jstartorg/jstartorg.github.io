  • Be a business user

The Templates

  • edit in anything for CSV, most commonly Excel
  • Templates can be downloaded from Company Setup Checklist
    • customized based on your company configuration
    • actual initial file is a binary XLS
    • must be saved to a CSV to import, so all the normal CSV caveats apply
    • can create our own XLSX template (see the Power Query session) just look out for any new required fields which will make the import fail
    • Pro-tip - using conditional formatting to highlight Required fields (or find and replace
  • first row is the Header - really the API name of the fields
  • second row column Description
    • Hashes = comment - applies to the whole row if in the first row cell, column if in the Header cell
    • has details like dependencies, length, required, editable
    • you can make your own note rows, or delete them entirely
  • Company Setup Checklist has a whole lot of options
  • An asterisk next to the object name means data can be UPDATED, not just inserted
  • Auto-sequencers = automatic ID for object, some times required
  • custom fields will always be at the end of all the columns
  • CASE SENSITIVE and space VALUES

SUPER INCONSISTENT VALUES

  • literally true/false and True/False and T/F are all Booleans and different across different fields ("just depended on the engineer who created it in the last 18 years")
  • can delete not-required columns (fields) from file
  • also going to test deleting the comment row - works fine
  • can move columns around no problem
    • except when you can't, became more clear just leave those things alone
  • API calls and PHP are a big thing - this is literally MyJ

Import Action

Steps for import

  • Imports trigger Smart Rules/Events
  • Most of the time things are in plain CSV, but check to see if UTF-8 is in play
  • You can import directly from the relevant page OR from the Company Setup Checklist
  • determine is it Top-Level or Entity level data
  • if it is a larger import, use Process Offline — really just always use Process Offline because it handles errors WAY better
  • It will not give any warning on double import of the same file/data; need to find Unique Field that will throw an error, probably custom field on GL Entry object
    • update: there is no such thing as a unique field in Intacct, so have to come up with another way to work with automated safe imports; maybe make staging object and then have some kind of smart event that hunts if it has already run
  • DOESN'T IMPORT ANYTHING AFTER BLANK ROWS unless it is a commented out row
  • there is no book on error codes

Fields for notes:

  • Doc - line item (she used this to hold the Batch number)
  • Description - GL Entry (formatted eg "8/2021 Monthly Thing")
  • Memo - line item
