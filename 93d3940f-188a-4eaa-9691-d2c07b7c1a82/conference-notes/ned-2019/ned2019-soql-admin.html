An Admin’s Guide to SOQL

SOQL = Salesforce Object Query Language

Use the SOQL to search your org’s SF data for specific information without running a report

Why Learn SOQL?

  • Provides access to more data than visible in the UI
  • Allows you to move up and down relationships
  • More robust and targeted extracts
  • No need for custom report types
  • More custom criteria/organization for extracts than reports
  • No need for separate applications to create extracts
  • Tools like SF Inspector or the Development Console give you access to the whole database in a couple of clicks
  • Keystrokes are always faster than clicks. SOQL allows you to scan using the keyboard instead of the mouse.
  • Query histories allow repeated requests to be run quickly

Situations to Use SOQL

  • Quickly comparing records/reviewing specific data points
  • Create a quick extract to ping a specified group of records
  • Review a specific group of parameters and outcomes across an extensive dataset
  • Conga solutions
  • Complex data loader extracts
  • Communicating with developers
  • Understanding code, flows, processes, and other SFDC functionality

How Do You Use SOQL?

Select function

  • Signifies a list of outputs (fields or data points) that the query should return
  • Data points can be found through relationships (_r instead of _c)

From function

  • Choose an object to pull data from
  • More objects than featured in the UI are available
  • Groups
  • Feed Tables

Where function

  • Filter criteria
  • Can use AND, OR, and () to provide advanced logic
  • Ex: WHERE Id = “00Q…”, Where custom_field__c = true,

Order By function

  • Order list by a certain field (e.g. LastName)

Other functions

  • SCOPE
  • LIMIT
  • OFFSET
  • GROUP BY
  • HAVING

SOQL in Action

Workbench (free)

  • Can click and select fields for functions and filters, or just type the query
  • Run query
  • Displays records at bottom

Salesforce Inspector (free)

  • Plugin
  • Show all data – show all data associated with record, even if not on page layout
  • Data export – build SOQL query
  • Can start typing and it will autofill
  • Export (copy to excel, csv, and json)

Limits to SOQL

  • Child to parent – have to start at the lowest child and work your way up. 5 levels (i.e. activity to child to classroom registration to classroom to account), 35 relationships

  • Parent to child – 1 level, 20 relationships (i.e. account to contact)
