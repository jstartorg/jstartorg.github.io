Formulas for the Everyday Admin

When creating a formula in a field:

  • Insert field button – gives you API field name and cross object references
  • Insert function button – when you scroll, SF pulls up syntactically correct version with example and link to online training

Using Basic Math to Simplify Complex Formulas

  • CASE – goes to field one time with list of questions and list of results based on the answer returned. Instead of using many ISPICKVAL functions, can convert each value to a number and do basic math (greater than, less than) to evaluate whether the current stage is less than the previous stage, for example
  • To get count of values in multi picklist field: IF INCLUDES statement for each value, return 1 if true and 0 if false, add the total

Easy, Dynamic Report Filters

To get one report for each user (owner, created by), that functions like “my tasks,”

  • Fields Referenced: CreatedById, $User.Id (current user).
  • Formula checkbox field: $User.Id = CreatedById (TRUE).
  • Can use it as list view filter criteria, report filter criteria. Can also do it for user role, user profile, any field that’s on the user record.

Dynamic month to date filter

  • Formula checkbox field DAY(CloseDate) <= DAY(TODAY))
    • DAY – evaluates date field, takes away month and year, and just returns numeric value of the day
    • TODAY – current date
  • Once you create the field, automatically calculates in every record in the org and is available in standard reports (have to manually add to custom report)
  • Can use as report filter, list view filter, report grouping

The Ultimate Parent Account

  • Use BLANKVALUE function to substitute the top parent account name with the next lowest parent account in the hierarchy if blank
  • Account formula field with text result:
    BLANKVALUE(Parent.Parent.Parent.Parent.Parent.Name,
    BLANKVALUE(Parent.Parent.Parent.Parent.Name,
    BLANKVALUE(Parent.Parent.Parent.Name,
    BLANKVALUE(Parent.Parent.Name,
    BLANKVALUE(Parent.Name)
  • Group report by ultimate parent account name to see all the revenue of parent org and all subsidiaries

The Hidden Magic of the Opportunity Stage Field

  • Have to assign default probability, forecast category, and whether it’s open, closed/won, or closed/lost
  • In reports, can see fields IsClosed and IsWon
  • In a nested IF statement, if it meets the first criteria, returns that value and is removed from the list:
    IF(IsWon – TRUE< “Won”,
    IF(IsClosed = TRUE, “Lost”,
    “Open”))
  • That way you can group by this field and don’t have to rebuild buckets for every report
  • When creating formula fields, check by reporting on both formula fields and results

The Power of One

  • Formula field on the object you want to count in report, number as result, no decimals
  • Name it after the object you’re creating it on
  • Formula: 1
  • Then every record of that object will have a 1
    • In report of opps with products, Total Records could mean opps, accounts, products, opp owners, etc
    • Instead, can summarize all power of one fields for all those objects
