Description

Each service or software we use has a license. Literally everything, every image, font, piece of data, everything has a license. Broadly speaking we have tried to make the most of open licenses/open source software where we will have the least overhead or legal concern, but obviously that only takes us so far.

Purchasing

As a note, it is best to start most purchases by going to TechSoup and searching for the desired resource. They have pre-arranged discounts with the stipulations already documented, making the process easier. Failing that, going to the service or software company directly and asking for a nonprofit discount is a good next step. Finally we can always just pay whatever they are asking, but it is always worth trying to get a discount as they can be significant. Understanding licensing is also at the core of Cost Management.

Services

Salesforce

Licensing is done on a per-active-user basis, with different license types allowing for different abilities within the system. We are currently using essentially two licenses types - the free Chatter license which just allows a user to authenticate (for CMs to use User Services/jstats), and the Sales Cloud (or Sales+Service Cloud) license which is the traditional full-feature license for staff to actually use Salesforce normally.

Microsoft/Office 365

Licensing is done on a per-user-per-application basis; for instance an affiliate site manager may have a free Office E1 license allowing them to use email and Teams, but not giving them Office desktop apps as those are to be provided by the site; whereas a corporate site manager will have an Office E5 license which does cost money but gives them the Office desktop apps.

Volume Licensing

We have participated in Microsoft's Volume Licensing program for many years, though less so as they move to heavily subscription based plans directly through Office 365. Management of the VLSC can be done through the Microsoft Admin Center or through the Volume Licensing Service Center using the o365.admin@jstart.org user account

Windows Server 2016

For a period of time we had access to some double digit number of Windows Server 2016 licenses, at which time we updated all of our servers from Server 2008. In the intervening time the license for that has disappeared from the portal, so it is not available going forward. This is no huge loss as we should be looking to implement either a newer Windows Server release or replacing the servers with Linux and potentially replacing Hyper-V with the open source hypervisor Proxmox

Windows 10 Enterprise

The Windows desktop experience most users are familiar with. The Enterprise license level allows us to implement several Group Policy features that are otherwise unavailable or more difficult to manage.
