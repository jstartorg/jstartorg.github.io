Shopify Sync

Sync Setup

Visit the Salesforce Sync app within Shopify to view setup (there are links to their documentation at the bottom of the page)

Salesforce Automations

The following automations update Shopify-related records in Salesforce:

  • On Save: Contact: For newly created records,
    • For records where Shopify Billing Company Name is blank, sets Shopify Customer Id on Contact's Household to the Shopify Customer Id of the Contact
    • Maps Shopify Email to either Personal Email field or Work Email field depending on domain
  • On Save: Account: For newly created records,
    • Sets appropriate Record Type and Name
  • On Save: Opportunity:
    • For newly created records, sets appropriate Opportunity Name and Stage
    • For records where Online Payment Tags is changed to include "site-" or "coll-", creates a Task for OpsTech queue to verify/clean Contact and Account data
    • For records where Shopify Total Discount is > 0 and there isn't already a Price Adjustment Opportunity Line Item in the amount of -(Shopify Total Discount), creates said Opportunity Line Item
  • On Save: Opportunity Line Item: For newly created records,
    • Sets the Opportunity Owner, Contact Owner, Account Owner, and Account Executive to the Default RFTR Record Owner found in the Jumpstart Setting custom metadata record (only for records where the current Owner is System Administrator/Account Executive is blank)

Data Maintenance

Regular Maintenance for Every Order

Every time an Opportunity is created from Shopify, take the following steps to verify/clean the associated Contact/Account data. Email salesforce@jstart.org if you need support.

  1. Confirm if order synced to correct Account. If incorrect, update Account on Opportunity, and move data from Shopify fields in System Information section on incorrect Account to the same fields on correct Account.
  2. Compare the customer information in the Purchase Details section on the Opportunity with the information in the Account Details section, and update the Account Details as needed.
  3. Confirm if order synced to correct Contact. If incorrect, update Primary Contact on Opportunity, and move data from Shopify fields in System Information section on incorrect Contact to the same fields on correct Contact.
  4. Compare the customer information in the Purchase Details section on the Opportunity with the information on the Opportunity's Primary Contact record, and update the Contact record if needed.
  5. If Shopify Billing Company Name on Opportunity is not blank and Contact does not have Affiliation with that organization, create one if needed.
  6. Check for duplicate Accounts, Contacts, and Affiliations related to the Opportunity, and merge as needed.

Sync Errors and Manual Resyncing

Error messages are sent to salesforce@jstart.org. If there is an error in syncing, or if you need to manually resync an order due to a change made in Shopify:

  • Fix any issues identified in the error message if applicable
  • Go to the Salesforce Sync app > Manually Sync Now
    • Send Shopify Data: Specific Shopify Orders
    • Shopify Object Ids: Enter the order id of the one that needs to be resynced (this can be found in the Shopify Order Id field on the Opportunity, or in the URL for the order in Shopify (the string of numbers after "orders/").
    • Click Send to CRM Now
  • Refresh the Opportunity record page in Salesforce to confirm the Amount matches the Shopify Total Amount and the sum of the Opportunity Line Items.

Anonymous Households

The sync tool will create an Anonymous Household when the order is placed for an individual rather than an organization, and it finds a Contact with a matching Shopify Customer Id or Email, but not an Account.

  • When it doesn't find a matching Contact:
    1. Sync tool creates Contact, sets Shopify Customer Id.
    2. If no Shopify Billing Company Name, Salesforce automation sets Shopify Customer Id on the Contact's automatically generated Household.
    3. Sync tool looks for Account with matching Shopify Customer Id, finds HH mentioned in step 2.
  • When it does find a matching Contact:
    1. Moves onto looking for matching Account, if none, creates one (even though Contact already has a HH). HH gets named as Anonymous because there is no Contact linked to it to get the name from.

To merge the duplicates,

  • Merge known duplicates in Duplicate Check
    • Go to Duplicate Check app > DC Search tab > Accounts tab
    • Search by Account Name or Primary Contact
    • Merge duplicate Accounts
    • Go to Opportunity record, click dropdown in top right, and click Refresh Name so that the Opportunity Name shows the real HH name rather than Anonymous
  • Run a Duplicate Check Job to find duplicates
    • Go to Duplicate Check app > DC Job > Add New Job
      • Job Name: Give the job a descriptive name, such as "Households from Shopify"
      • Select Object: Account
      • Scenario: Same Primary Contact
      • + Filter
        1. Record Type Name starts with Household
        2. Shopify Customer Id not equal to [leave value blank]
      • Filter Logic: Leave blank
      • Compare with: Set filter
        1. Record Type Name starts with Household
      • Begin the job by clicking DC Local if you have the app on your computer, or Start if not
    • Merge and discard the duplicates in the job as applicable
    • Refresh any Opportunity Names that need to be refreshed

Can also monitor duplicates in the Duplicate Households Created by Shopify report
